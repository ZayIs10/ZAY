# Gen Z Capital — Faceless Reel Creation Playbook

Complete strategy + execution guide for producing viral faceless wealth Reels.
This is the SOP for the Reel side of the brand. Every Reel must follow this doc.

---

## Why Reels (Not Static Posts)

Instagram's 2026 algorithm only pushes Reels to non-followers on new/small accounts.
Static posts get shown to your existing followers — if you have 0, nobody sees them.
Reels are the **only free distribution channel** for a zero-follower account.

**Performance benchmark:** Faceless wealth accounts pull 50K–500K views per Reel.
Real examples: @wealth.simple.tips, @cryptonary, @thepositiveminds, @coffeecontracts, @factsdailyy.
These accounts grew 0 → 100K followers in 6–12 months with Reels only. No ads. No face.

---

## The Algorithm (2026) — Ranking Signals in Order of Weight

For Reels specifically:

1. **DM Shares** — Most heavily weighted (3–5x more than likes)
2. **Watch Completion** — Watching past 75% of the video
3. **Saves** — "Worth referencing later"
4. **Comments** — Engagement depth
5. **Story Shares** — Resharing to stories
6. **Likes** — Weakest signal

**Implication:** Every Reel must be designed to be DM'd, saved, and watched to completion.
Optimize for those three. Ignore likes.

## The 5 Non-Negotiable Rules

Every successful faceless wealth Reel follows these. No exceptions.

### 1. Length: 15–30 seconds
Sweet spot for completion rate. A 22-second Reel watched to the end beats a 60-second
Reel watched 40%. Algorithm rewards completion %, not length.

### 2. Text appears BEFORE voiceover (0.5s earlier)
Eyes register text faster than ears process speech. Putting text on screen first
holds the viewer 0.5s longer — enough to clear the 3-second danger zone.

### 3. The "Bridge" Hook Formula
Every Reel opens with this structure:

> **"[Unusual claim in 7-9 words]... and here's why."**

Examples:
- "99% of people will never own a home... and here's why."
- "This one habit makes millionaires... most ignore it."
- "AI just killed the 9-to-5... here's what replaces it."

The "...and here's why" forces the brain to keep watching for resolution.

### 4. Subtitles are mandatory
85% of Instagram users watch with sound off. Subtitles increase average watch time
by 25%. Every line of voiceover must appear as bold subtitle text.

### 5. Trending audio at 20% volume under the voiceover
Reels using trending audio get 30% more reach (Instagram surfaces them in audio-based
discovery). Keep AI voiceover at 100%, trending audio at 20% as background energy.

## The 5-Beat Structure (Frame-by-Frame)

Every Reel = exactly 30 seconds = 5 beats.

### Beat 1: HOOK (0–3s)
- **Visual:** Cinematic b-roll (city skyline at night, money close-up, server room) — slow zoom
- **Text:** Bold claim. Anton font, white. Big number in neon green (`#39FF14`)
- **Voiceover:** "[Shocking number]% of [audience] will [bad outcome]."
- **Audio:** Bass-heavy cinematic music kicks in

### Beat 2: PROBLEM (3–10s)
- **Visual:** Cut to second cinematic shot (dim desk lamp, hands counting bills)
- **Text:** "THE REASON?" → fades to "THEY DON'T KNOW THIS."
- **Voiceover:** "The reason isn't [X]. It's not [Y]. They just don't know this one thing."
- **Open loop created** — viewer must keep watching

### Beat 3: INSIGHT (10–20s)
- **Visual:** One of your existing DALL-E cinematic images (Bloomberg terminal, etc.)
- **Text:** 3 short lines, max 4 words each:
  - Line 1: White
  - Line 2: Neon green (key number/word)
  - Line 3: White
- **Voiceover:** Concrete, specific, actionable insight (max 12 words)

### Beat 4: PROOF (20–27s)
- **Visual:** Solid black background
- **Text:** ONE massive number in neon green, fills the screen. Small label below in white.
  Number "counts up" from $0 → final amount in 2 seconds
- **Voiceover:** State the number. Period.
- **This is the screenshot moment** — viewers save the Reel here

### Beat 5: CTA (27–30s)
- **Visual:** Cut back to cinematic shot. Logo in corner.
- **Text:** "SAVE THIS." then beneath: "FOLLOW @genzcapital"
- **Voiceover:** "Save this. Follow for more."
- **Animation:** Neon green divider line animates across screen

Stay 100% consistent with these rules — this is what makes the brand recognizable.

| Element | Spec |
|---|---|

| Headline font | Anton (bold, ALL CAPS, condensed) |

| Text colors | White `#FFFFFF`, Neon green `#39FF14`, Gray `#A0A0A0` |
| Background | Dark, cinematic, single dramatic light source |
| No people's faces | Silhouettes or backs only |

| Logo placement | Top-center on cover frame, corner on CTA frame |
| Divider lines | Neon green, both sides of logo |

| Mistake | Why |

## Manual Workflow (15 min per Reel)

For each Reel, follow these steps in order:

1. **Get content from automation**
   - Run `publisher/post_generator.py` on a "Ready" topic from Google Sheets
   - From the logs, copy the `reel_script` field (5 punchy lines)
   - Save 3–4 of the cinematic DALL-E images from `assets/images/generated/`

2. **Generate voiceover**
   - Open ElevenLabs → paste reel_script
   - Voice: "Adam" (deep, serious male)
   - Settings: stability 50%, clarity 75%
   - Download MP3

3. **Find b-roll** (optional, can skip if using DALL-E images only)
   - Pexels search: "city skyline night," "trading floor," "money close-up," "server room"
   - Download 2-3 vertical clips, 5-10 seconds each

4. **Build the Reel in CapCut**
   - New 9:16 project
   - Drop in: cinematic image #1 (3s) → b-roll (7s) → cinematic image #2 (10s) → black bg with stat (7s) → cinematic image #3 (3s)
   - Apply slow zoom to each image (Ken Burns effect)
   - Add voiceover audio track on top
   - Add text overlays — Anton font, white + neon green, large
   - Find a trending finance audio in CapCut's audio library → add at 20% volume
   - Auto-generate captions (CapCut → Captions → Auto Captions) → style with white box, Anton font

5. **Export**
   - 1080×1920, 30 fps, high quality
   - Save MP4 to `assets/reels/`

6. **Post to Instagram**
   - Upload as Reel (NOT as a regular post)
   - Caption: paste the `post_caption` from automation (already has DM-share CTA)
   - Hashtags: see hashtag list below
   - Cover frame: select frame at 0.5s (the hook moment)
   - Post

## Phase 2 — Automated Reel Pipeline (HyperFrames)

Phase 2 is now built. The renderer is `publisher/reel_generator.py` driving
[HeyGen HyperFrames](https://github.com/heygen-com/hyperframes) — an open-source
(Apache 2.0), HTML→MP4 framework designed for AI agents (this is the toolchain
from the Nick Harrick "Claude Just Destroyed Every Video Editing Tool" video).

### Why HyperFrames over moviepy

- **Brand fidelity:** the reel layout is HTML/CSS — same Anton font, same
  `#39FF14` neon green, same 1080×1920 stage as our static posts. moviepy's
  text rendering would have been a fight.
- **Animation:** GSAP timelines give us the slow Ken-Burns push-in,
  count-up numbers, and crossfades from the 5-beat spec for free.
- **Determinism:** same input → identical MP4 output, suitable for CI.
- **Free:** no per-render fees, no API quota, no cloud round-trip.

### Project structure

```
reels/
  index.html       ← the active composition (rewritten by reel_generator.py per run)
  styles.css       ← brand-locked CSS (Anton, #39FF14, vignettes, layouts)
  script.js        ← GSAP timeline (Ken-Burns, count-up, crossfades, dividers)
  hyperframes.json ← project config
  meta.json        ← project id

publisher/
  reel_generator.py ← orchestrator: Sheets → spec → HTML → render → MP4

scripts/
  with-ffmpeg.js   ← shim that puts the npm-installed ffmpeg/ffprobe on PATH

assets/
  reels/<slug>.mp4 ← rendered output goes here
```

### Toolchain (one-time setup)

| Tool             | How                                                       |
| ---------------- | --------------------------------------------------------- |
| Node.js >= 22    | already on this machine                                   |
| FFmpeg + FFprobe | `npm install --save-dev @ffmpeg-installer/ffmpeg @ffprobe-installer/ffprobe` (portable, in `node_modules/`) |
| Chrome Headless  | `npx hyperframes browser ensure` (one-time, ~100 MB)      |
| HyperFrames CLI  | `npm install --save-dev hyperframes`                      |
| Claude Code skills (optional) | `npx skills add heygen-com/hyperframes` — enables `/hyperframes`, `/hyperframes-cli`, `/gsap` slash commands. Run this manually; the agent harness blocks installing external skills automatically. |

### Daily workflow

1. **Generate the day's static post first** (existing pipeline):
   `python publisher/post_generator.py` → produces DALL-E images in
   `assets/images/generated/` and a `reel_script` log entry.
2. **Render the reel from the next "Ready" sheet row**:
   `python publisher/reel_generator.py`
   This rewrites `reels/index.html`, lints, then renders to
   `assets/reels/<topic_slug>.mp4`.
3. **Smoke-test without Sheets / GPT**:
   `python publisher/reel_generator.py --sample` (uses hard-coded sample copy)
4. **Preview interactively** before committing render time:
   `npm run reels:preview` → studio editor in browser, live reload.
5. **Upload manually to Instagram Reels.** Posting via the Graph API is
   blocked on the `instagram_content_publish` permission, same as static
   posts — we resume automated upload once that's resolved.

### npm script shortcuts

| Command                  | What it does                                |
| ------------------------ | ------------------------------------------- |
| `npm run reels:doctor`   | Verify ffmpeg/ffprobe/Chrome are reachable. |
| `npm run reels:preview`  | Live-reload preview in browser.             |
| `npm run reels:lint`     | Validate the composition (always run after edits). |
| `npm run reels:render`   | Render `reels/` to `renders/<name>.mp4` (2 Chrome workers, stable on this machine). |

### Voiceover (ElevenLabs) — not yet wired

`reel_generator.py` already accepts a `voiceover_src` field on `ReelSpec`,
plays it on track 1, and mixes background music on track 2 at 20% volume
(rule #5 of this playbook). Connecting ElevenLabs is a 30-line addition:
`POST /v1/text-to-speech/{voice}` with the `reel_script` field already
emitted by `post_generator.py`. Add `ELEVENLABS_API_KEY` to `.env` when
ready and we'll wire the call.

### Render performance — Windows note

This machine's parallel renderer crashed at 8 workers (the auto default)
with `Page.captureScreenshot: Target closed` errors. Pinned to `--workers 2`
in `package.json`. A 30-second 9:16 reel renders in ~2–3 minutes at draft
quality, ~5–8 minutes at standard.
